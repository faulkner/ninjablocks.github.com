Block

Quick Access

To get you up and running quickly, we've built in a virtual block token, which you can enable (and revoke), in the account settings page. This block token will allow you to create arbitrary devices under your account without activating or claiming.

Use your virtual block token exactly as you would a standard block token, i.e. pass it as an http header named X-Ninja-Token.

Resource Description
GET block/:nodeId/activate Returns the block token after a user has claimed the block. This token should be used with all subsequent requests. Your nodeId is a 12+ character identifier, ideally a unique serial number.
POST block Attempt to claim an unclaimed block.
GET block/:nodeId/commands Long GET that will response with that block's commands for the duration of the request.
POST block/:nodeId/heartbeat Send a heartbeat from this block.
POST block/:nodeId/data Send a 'data' event from this block. Typically 'data' events will be instantly actioned, as opposed to heartbeats which may be queued.

 

GET block/:nodeId/activate

Returns the block token when an authenticated user has claimed the block. This token should be used with all subsequent requests. This should be implemented as a long GET.

Parameters
none

Notes

  • nodeId should be a unique string at least 12 characters in length, ideally a hardware serial number accessible by the user.
  • This URI does not currently require authentication credentials.
  • A response will be returned only after a user has successfully executed the POST to claim a block. If the response ends without a body, the request should be retried.

Resource URL

https://api.ninja.is/rest/v0/block/:nodeId/activate

Sample JSON response

{
"token": "1c0c8929-3318-411d-96e0-36fb7872ea5c"
}

Curl Example

Await activation of a block with nodeId 123456789ABC

curl -H 'Content-Type: application/json' \
   -X 'GET' \
   -i https://api.ninja.is/rest/v0/block/123456789ABC/activate

 

POST block

Attempt to claim an unclaimed block.

Parameters
nodeid
string
Required		 The ID of the block you are attempting to claim. This block must be currently awaiting activation. ID's have a minimum length of 12 characters.
Example: "ASTEALTHYNODE01"

Notes

  • Note that a result of 1 indicates the claim attempt was received successfully NOT that the claim was successful (this will change in V1 of the API). The first block requesting activation with a matching nodeid within 5 minutes will be given a token linking it to the user making this request.

Resource URL

https://api.ninja.is/rest/v0/block

Sample JSON response

{
"result": 1,
"error": null, 
"id": 0
}

Curl Example

Attempt to claim block 123456789ABC on behalf of the user identified by their user access token

curl -H 'Content-Type: application/json' \
     -H 'X-Ninja-Token: ' \
     -X 'POST' \
     -i https://api.ninja.is/rest/v0/block?user_access_token=YOUR_ACCESS_TOKEN \
     -d '{"nodeid":"123456789ABC"}'

 

GET block/:nodeId/commands

Long GET that will respond with that block's commands for the duration of the request.

Parameters
none

Notes

  • X-Ninja-Token is required in the headers of this request, and corresponds to the token acquired via the GET block/:nodeId/activate route.
  • nodeId must be at least 12 characters in length.
  • Commands are separated by the newline character \n.
  • This request is long lived, and should be reinitialized if the connection is severed.

Resource URL

https://api.ninja.is/rest/v0/block/:nodeId/commands

Sample JSON response

{"NODE_ID":"ASTEALTHYNODE01","DEVICE":[{"GUID":"ASTEALTHYNODE01_0_0_1000","G":0,"V":0,"D":1,"DA":"FFFF00"}]}
{"NODE_ID":"ASTEALTHYNODE01","DEVICE":[{"GUID":"ASTEALTHYNODE01_0_0_11","G":0,"V":0,"D":1,"DA":"001100110011000011100101"}]}
{"NODE_ID":"ASTEALTHYNODE01","DEVICE":[{"GUID":"ASTEALTHYNODE01_0_0_1000","G":0,"V":0,"D":1,"DA":"FFFFFF"}]}
{"NODE_ID":"ASTEALTHYNODE01","DEVICE":[{"GUID":"ASTEALTHYNODE01_0_0_11","G":0,"V":0,"D":1,"DA":"100010100010101011010110"}]}

 

POST block/:nodeId/heartbeat

Send a heartbeat from this block.
Important: the request body takes the form of an array of objects, whose parameters are outlined below.

Heartbeat events are intended to be values to be analyzed in due course. For example, a temperature reading should be part of a 'heartbeat' event, whereas a button press should be a 'data' event.

Each Array Element Parameters
GUID
string
Required		 The GUID of the device reporting.
Example: "ASTEALTHYNODE01_0_0_11"
G
string/number
Required		 The port number the device is located on.
Example: 0
V
string/number
Required		 The Vendor ID of the device.
Example: "0"
D
string/number
Required		 The device ID.
Example: 11
DA
string/number
Required		 The data value that this device is reporting.
Example: "-1"

Notes

  • X-Ninja-Token is required in the headers of this request, and corresponds to the token acquired via the GET block/:nodeId/activate route.

Resource URL

https://api.ninja.is/rest/v0/block/:nodeId/heartbeat

Sample JSON response

{
"result": 1,
"error": null, 
"id": 0
}
