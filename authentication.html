Authentication

We are currently working on creating OAuth 2 endpoints for authentication & authorization, however at present this is not yet implemented.

Until then, we ask that developers use our regular login endpoint which will return a 'ninja.sid' session token. This token should be used as in the 'Cookie' header for all API endpoints, and will allow access to all of the authenticated user's resources.

Important: the Accept header should always be set to 'application/json' if you are expecting json, otherwise you might receive a 302 if the session has timed out. This is only temporary until our OAuth2 provider service in in place.

Resource URL

https://api.ninja.is/signin

Example

$ curl -H 'Content-Type: application/json' \
 -H 'Accept: application/json' \
 -d '{"email":"email@example.com","password":"correctpassword"}' \
 -i https://api.ninja.is/signin | json
