Module [DRAFT]

Modules are pieces of software that interface between the Ninja platform and hardware devices. Think of them as hardware drivers that run inside our client. Our client is written in Node.js, therefore so are our modules.

A module is made up of a constructor responsible for registering devices that speak certain protocols.

The protocol they talk is defined by their device_type which itself is derived from the device's, V, and D. (See core concepts for more on that).

The constructor/device both have methods/properties/events associated with them that provide an interface the Ninja client can understand.

Writing a Module Quick Start Guide [WIP]

Step 1 - pick the device(s) you'll be working with

Soon you will be able to register as your own vendor (thus getting your own V) and manage your own device list, however until then we have a big list of generic devices that you can choose from. The list is located here.

Step 2 - look up its protocol

Each device type has a different protocol that must be adhered to so that other developers can understand how to talk to your devices. The list of protocols we have written (so far) is located here.

Note: these protocols are meant to be written by the community. If you want to contribute, submit a pull request to this documentation, which itself is just a GitHub repository located here.

Step 3 - work on our client's develop branch (temporary).

We wrote a How-To on that here.

Step 4 - create a module scaffold

We have a beta CLI tool to help with this, it is located here.

Step 5 - modify the device with the vendor (V) and device (D) IDs from Step 1

If you used our CLI from step 4, this will be in PATH_TO_NINJA_CLIENT/ninja_modules/<module name>/lib/device.js.
