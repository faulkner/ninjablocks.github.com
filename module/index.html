Module [DRAFT]

Modules are pieces of software the interface between the Ninja platform and hardware devices. Think of them like hardware drivers that run inside of our client. Our client is written in Node.js, therefore so are our modules.

A module is made up of a constructor responsible for registering devices that speak certain protocols.

The protocol they talk is defined by their device_type which itself is derived from the device's G, V, and D. (See core concepts for more on that).

The constructor/device both have methods/properties/events associated with them that provide an interface the Ninja client can understand.
